## Bulunan Hatalar ve Çözümleri

### 1. Concurrent Requests Ollama'yı Kitliyor
- **Belirti**: 3 worker aynı anda qwen2.5-coder:7b'ye istek atınca GPU overload → 
  hepsi HTTP read timeout (120sn) → fallback'e düşer
- **Kök Neden**: Ollama 7B model tek GPU'da sıralı işler, 3 concurrent istek
  queue'da birikip timeout limitini aşar
- **Çözüm (v8)**: 
  - `Semaphore(1)` ile 1 thread sınırlandırması
  - Health check: `requests.get("/api/tags", timeout=5)` — Ollama ayakta değilse atla
  - v7'de semaphore vardı ama 120sn timeout + concurrent queue yine öldürüyordu

### 2. Dict Race Condition
- **Belirti**: `RuntimeError: dictionary changed size during iteration`
- **Kök Neden**: ThreadPoolExecutor içinde tüm thread'ler aynı anda 
  `cache` dict'ine yazıyor. `json.dump(cache, f)` iterasyon sırasında 
  başka thread ekleme/silme yaparsa crash
- **Çözüm**: `json.dump(dict(cache), f, ...)` — snapshot al

### 3. Generic Skill İsimleri ("hermes-agent")
- **Belirti**: 50+ skill "hermes-agent" ismiyle kaydedilmiş
- **Kök Neden**: OpenRouter "name: hermes-agent" döndürünce üzerine yazıyor
- **Çözüm (v8)**: `extract_name()`'de `n2 != 'hermes-agent'` kontrolü

### 4. API Sırası
- **Karar**: OpenRouter GPT-4o-mini en hızlı (10-15 sn/video) ve en kaliteli
  → Öncelikli yapıldı (v6'da Gemini önceydi)
- Gemini API key geçerli ama hiç response alınamadı (format sorunu olabilir)
- Ollama sadece yedek
