### Konu 1/12: **Python Decorators**

Decorators, fonksiyonları başka bir fonksiyona argüman olarak geçirerek işlevselliklerini genişleten yüksek seviyeli fonksiyonlardır. Bir decorator, bir fonksiyonu alır, etrafını sarar ve geliştirilmiş halini döndürür.

---

### 🔑 Öne Çıkan Kod Örnekleri

**1. Temel Decorator Yapısı — Fonksiyon Öncesi/Sonrası İşlem**
```python
def my_decorator(func):
    def wrapper():
        print("Before function call")
        func()
        print("After function call")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()
# Output: Before function call → Hello! → After function call
```

**2. Logging Decorator — Gerçek Hayat Kullanımı**
```python
import logging
def log_decorator(func):
    def wrapper(*args, **kwargs):
        logging.info(f"Calling {func.__name__} with {args}, {kwargs}")
        result = func(*args, **kwargs)
        logging.info(f"{func.__name__} returned {result}")
        return result
    return wrapper

@log_decorator
def add(a, b):
    return a + b
```

**3. Performance Monitoring Decorator**
```python
import time
def measure_time(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

### ✅ Best Practices

1. **Decorator'lar fonksiyonun temel yapısını değiştirmemeli** — sadece ek davranış katmalı
2. **Küçük ve öz olmalı** — tek bir sorumlulukları olmalı
3. **Anlamlı isimlendirme** — kod okuyucularının decorator'ın ne yaptığını hemen anlaması kolay olmalı

### ⚠️ Yaygın Hatalar
- Fonksiyon argüman listesini `wrapper(*args, **kwargs)` ile doğru geçirmemek
- Decorator'un döndürdüğü fonksiyonun, orijinalin argüman/return tipleriyle uyumsuz olması
